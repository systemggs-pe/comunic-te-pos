# GGS brand architecture

## Brand hierarchy

- `GGS`: parent company and corporate ecosystem.
- `GGS SYSTEMS`: commercial software brand for SaaS products and business platforms.
- `GGS CODE`: engineering and development division responsible for architecture, implementation, security and delivery.
- `COMUNIC@TE`: operational SaaS product within the GGS SYSTEMS portfolio.

## Corporate wording

Primary:

```text
COMUNIC@TE operates under GGS SYSTEMS, a GGS software ecosystem engineered by GGS CODE.
```

Footer:

```text
COMUNIC@TE, GGS SYSTEMS and related software assets are part of the GGS ecosystem.
```

Legal:

```text
GGS SYSTEMS is the commercial software brand. GGS CODE provides engineering, development and delivery. GGS is the parent company.
```

Short badge:

```text
GGS ecosystem
```

Technical attribution:

```text
Engineered by GGS CODE
```

## Visual rules

- Use `COMUNIC@TE` as the product name in primary UI surfaces.
- Use `GGS SYSTEMS` as the platform/software brand in secondary labels.
- Use `GGS CODE` only as technical attribution, engineering badge or copyright context.
- Use `GGS` sparingly as the parent company, not as a repeated decorative mark.
- Keep all brand mentions quiet, small and systematic inside product UI.
