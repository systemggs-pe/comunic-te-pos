# Deployment and DevOps

## Overview

The project is designed to deploy as:

- Frontend static assets generated by Vite.
- Netlify Functions for backend API.
- Firestore as database.
- Firebase Auth as identity provider.
- Firestore rules and indexes deployed through Firebase CLI.

There is also a Firebase Hosting configuration in `firebase.json`, but the README and Netlify configuration identify Netlify Functions as the canonical backend runtime.

## Local Requirements

Required:

- Node.js compatible with Vite 7 and React 19.
- npm.
- Firebase project access.
- Netlify site access for production functions.
- API credentials for RENIEC and Gemini if testing those features.

Recommended:

- Firebase CLI for rules/index deployment.
- Netlify CLI for local function parity if needed.

## Installation

```bash
npm install
```

Create local environment file:

```bash
cp .env.example .env
```

Populate `.env` with local values. See [Configuration](configuration.md).

## Local Development Modes

### Full local app with local API adapter

```bash
npm run dev
```

This starts:

- Local API adapter at `http://127.0.0.1:3001` by default.
- Vite frontend with `/api` proxied to local API.

Implementation:

- `scripts/dev-local.mjs`
- `scripts/local-api.mjs`
- `vite.config.js`

### Frontend only

```bash
npm run dev:vite
```

Use this when API calls are not required or `VITE_BACKEND_BASE_URL` points to a deployed backend.

### API only

```bash
npm run dev:api
```

### Netlify local functions

```bash
npx netlify dev
```

This uses Netlify's local function runtime.

## Netlify Deployment

Configuration:

```toml
[build]
  command = "npm run build"
  publish = "dist"

[functions]
  directory = "netlify/functions"
  node_bundler = "esbuild"
```

API redirects:

```text
/api/reniec -> /.netlify/functions/reniec
/api/analizarCajaGemini -> /.netlify/functions/analizarCajaGemini
/api/clientes -> /.netlify/functions/clientes
/api/legalConsent -> /.netlify/functions/legalConsent
/api/registros -> /.netlify/functions/registros
/api/ventas -> /.netlify/functions/ventas
```

SPA fallback:

```text
/* -> /index.html
```

## Firebase Deployment

Firestore rules and indexes are configured in:

- `firestore.rules`
- `firestore.indexes.json`
- `firebase.json`

Deploy rules/indexes:

```bash
npx firebase deploy --only firestore
```

Do not deploy Firebase Functions unless the architecture changes.

## Environment Variables in Production

Configure in Netlify:

- `FIREBASE_SERVICE_ACCOUNT`
- `FIREBASE_API_KEY`
- `ALLOWED_EMAILS`
- `ALLOWED_ORIGINS`
- `RENIEC_TOKEN`
- `GEMINI_API_KEY`
- `GEMINI_MODEL`
- `VITE_BACKEND_BASE_URL`

For frontend and functions hosted in the same Netlify site, `VITE_BACKEND_BASE_URL` can stay empty.

## SSL and Domains

Netlify and Firebase Hosting both provide managed HTTPS. Production domain hardening should include:

- Custom production domain.
- HTTPS enforced.
- HSTS header.
- Correct `ALLOWED_ORIGINS` with only production domains and required deploy previews.
- Legal sitemap/canonical URLs updated to the production domain.

## Backups

Current backup mechanism:

- Manual backup button in app exports loaded and complete paginated `registros` and `ventas` plus currently loaded `clientes` and `equipos`.

Limitations:

- Backup is user-triggered.
- Backup is downloaded to browser.
- There is no automated scheduled backup.

Enterprise recommendation:

- Scheduled Firestore export to Cloud Storage.
- Retention policy.
- Recovery runbook.
- Periodic restore testing.

## Monitoring

Current state:

- Console logging in frontend and local scripts.
- No structured monitoring/alerting integration.

Recommended:

- Netlify function logs with request IDs.
- Error reporting such as Sentry.
- Provider error rate dashboards.
- Firestore usage dashboards.
- Alerts for auth failures, API spikes, function errors and provider failures.

## Production Release Checklist

Before production:

- Run `npm run lint`.
- Run `npm run build`.
- Run tests once implemented.
- Run `npm audit`.
- Verify Netlify env vars.
- Verify Firestore rules deployed.
- Verify legal entity, tax data, domicile, processor agreements and retention schedules are confirmed for production.
- Verify allowed origins.
- Verify Google Auth authorized domain.
- Verify RENIEC and Gemini with test accounts.
- Verify PDF generation on desktop and mobile.
- Verify camera scanner after Permissions-Policy changes.
- Verify backup/export workflow.
